# ai_transcript — ASL gesture training

Train and test two gesture models used for live ASL recognition:

| Model | Algorithm | Input | Output artifact |
|-------|-----------|-------|-----------------|
| **Static** | Random Forest (+ scaler) | Single-frame hand landmarks (63-D) | `static_model.joblib` |
| **Motion** | GRU (PyTorch) | 10-frame motion sequences (147-D/frame) | `motion_model.pt` |

Static covers **fingerspelling letters** (A–Z; your dataset may omit **J** if no samples were collected). Motion covers **26 conversational signs** (HELLO, HELP, WATER, YES, …) defined in `build_train_manifests.py` → `TRAIN_MOTION_LABELS`.

## Full pipeline (train → test → export)

Run from this directory with the venv activated:

```powershell
cd ai/training/ai_transcript

# 0) Verify dependencies (optional)
python verify_setup.py

# 1) Optional: trim dataset to manifest-only files
python cleanup_dataset.py

# 2) Rebuild manifests from raw_custom (after new captures or cleanup)
python build_train_manifests.py

# 3) Train BOTH models (default quality gates)
python run_pipeline.py --mode both

# 4) If train already passed, regenerate PNGs only
python export_train_images.py --mode both

# 5) Live tests (manual — webcam)
python test.py
python test_motion.py

# 6) Optional: copy models into worker for the app
python run_pipeline.py --mode both --export-runtime
# or:
python export.py --strict
```

Shorthand after data is stable:

```powershell
python run_pipeline.py --mode both --rebuild-manifests
python verify_setup.py
```

## Quality gates

Training **fails** unless each model passes (override with `--allow-failed-gate`):

| Check | Default |
|-------|---------|
| Macro F1 (test split) | ≥ 0.70 |
| Min test support per class | ≥ 1 |
| Max off-diagonal confusion rate | ≤ 0.45 (only classes with **≥ 3** test samples) |

Inspect confusions: `python analyze_confusion.py --task motion`

Reports: `artifacts/gesture/train_report/*_train_report.json` → `"quality_gate": { "passed": true }`.

## Recommendations / known limits

1. **Letter J** — Static training uses only labels present in `raw_custom/static/`. Add `dataset/raw_custom/static/J/` samples and retrain for full A–Z.
2. **Small motion classes** — e.g. `OK` may have few samples; collect more if live accuracy is weak.
3. **`labels_v1.json` motion list** — Long care-vocabulary list is reference only; **training** uses `TRAIN_MOTION_LABELS` in `build_train_manifests.py` (26 signs).
4. **`live_sessions/`** — Debug output from motion tests; safe to delete; not required for submission.
5. **`build_manifest.json`** — Updated per `train.py` run; last run with `--mode both` should list both `static` and `motion` blocks.
6. **MediaPipe model** — First webcam run downloads `models/hand_landmarker.task` automatically.
